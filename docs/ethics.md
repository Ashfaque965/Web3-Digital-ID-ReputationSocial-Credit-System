# Important Considerations

## Strengths
- Transparent, immutable scoring
- User-controlled identity (SSI)
- Programmable incentives

## Risks
- Potential for exclusionary "social credit" mechanics
- Surveillance if oracles feed too much behavioral data
- Central issuers can still control credential validity
- Score manipulation / gaming

## Recommendations
- Make participation **opt-in**
- Use zero-knowledge proofs for privacy
- Implement appeal mechanisms
- Avoid single global score — use contextual multi-scores
- Regular audits and community governance
